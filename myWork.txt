TESTING APPROACH & ARCHITECTURE: C# CALCULATOR APP

1. PROJECT STRUCTURE
CalculatorAppSolution/
├── CalculatorApp.sln                     # Main solution file
├── src/
│   └── CalculatorApp/
│       ├── CalculatorApp.csproj          # Main application project
│       ├── Interfaces/
│       │   ├── ICalculator.cs            # Pure calculation contract
│       │   ├── IHistoryRepository.cs     # Persistence contract
│       │   └── ICalculatorService.cs     # Orchestrator contract
│       ├── Models/
│       │   └── CalculationRecord.cs      # Record type for history
│       ├── Services/
│       │   ├── Calculator.cs             # Pure math implementation
│       │   └── CalculatorService.cs      # Orchestrator implementation
│       └── Repositories/
│           └── XmlHistoryRepository.cs   # Real XML I/O implementation
└── tests/
    └── CalculatorApp.Tests/
        ├── CalculatorApp.Tests.csproj    # Test project (References CalculatorApp)
        ├── UnitTests/
        │   ├── CalculatorTests.cs        # Pure math logic tests (ICalculator)
        │   └── CalculatorServiceTests.cs # Orchestration tests with Moq (ICalculatorService)
        ├── IntegrationTests/
        │   └── XmlHistoryRepositoryTests.cs # Real File I/O tests (IHistoryRepository)
        └── TestResults/                  # Output directory for test reports

2. PROJECT DEPENDENCIES
- Framework: .NET 8.0 (or current)
- Main App: Minimal dependencies (System.Xml.Linq for XML handling)
- Test App: 
  - xunit: Primary testing framework
  - xunit.runner.visualstudio: Test explorer support
  - Moq: Mocking library for isolating the orchestrator
  - FluentAssertions: For highly readable assertions
  - Microsoft.NET.Test.Sdk: Build tools for testing
